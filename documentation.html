Description

The aim of the project is to create a software framework, which will enable web developers to use their skills and experience in web development and to create native applications with HTML5 content.
The basisi of the native container provided by the framework is a WebView, which executes the HTML5 contents and a native API which is responsible for handling request for native functionalities.

The HFH API provides the following:

  • native SQLite database
  • access to disk storage
  • Camera - photos and video
  • Motion Sensors
  • Contacts
  • Native views


Architecture

  1. View layer - common to all target platforms.
  2. Optional layer - common to all target platforsm.
  3. Platform specific code - used for tasks that cannot be accomplished in Javascript or are not optimal, or slow when done with Javascript.
The presented model can be viewed as an MVC model:
1 - View
2 and 3 - Model and Controller
We accomplish reuse of 1 and 2 throught out all target platforms.

How to use the HFH

Tutorial on using the HFH

Use the template project HFH and rename the project. In the left side of the Project explorer select projects derive the Detail editor and change the name of the project. Change the name of the directory in which the project and again in Detail editor indicating the location of the project to point to the renamed directory. In Project Settings choose each page and use the search function by searching the old name of the project and the results change it to the new name you have chosen. Clear the project and build if the build passes without errors renaming was successful. If an error occurs then we forget to rename some of the resources, threfore repeat each step to find what you missed.
Web content such as HTML pages, JavaScript files, images and other resources is located in the wwwroot directory
Delete the content of the wwwroot directory and copy your content in this directory.

In "CDVViewController.m" file add the methods from BridgeVCHfh.m file from the mark:

#pragma mark BirdgeListener Protocol

Add the following lines in the "CDVViewController.m" file:

4.In the "__init" method set listener for mBridgeHelper to self:

    mBridgeHelper.bridgeListener = self;
